Adding headings

Here is some text under the first subheading. Headings help define structure and hierarchy of content. They should be used consistently to plan out the website. Don't SKIP a series of headings (i.e. always go in order from h2, h3, ... and don't jump to hx+2).

Headings are one of the best ways to control document outlines.

Using Paragraphs

Use the paragraph tag (⟨p⟩) for each block or paragraph of text.

Every new paragraph should have its own set of tags.

Line Breaks

The line break tag (⟨br⟩) makes a new line without a new paragraph, known as a "soft return". These are inline (can be used within headers an paragraphs and do NOT require a closing tag).

Section vs. Div

The section tag denotes thematically grouped content to be put in a document outline, whereas the div tag does not convey any meaning and is generally used for styling or scripting purposes.

All sectioning tags (i.e. section, aside, article...) will generate headers in the outline. A sectioning tag that does not have a header will be labelled as Untitled Section

The Strong Tag
This looks just like the bold tag... but it actually has some meaning. This will affect screen readers and other interpreters. Originally the b tag was removed, but HTML5 brought it back because there could be cases where you wanted to bold something but not interpret it strongly.
The Italic Tag
This i tag makes text italicized.
The Emphasis Tag
This looks just like the i tag above, but the text is actually emphasized more than the text around it symantically. Although this tag is usually rendered in italics, it shouldn't be used in place of the italics tag, rather it should be used for text that needs to be emphasized for some reason.

Essentially, one set of tags is presentational, while the other is logical

Named Character Entities
Use & followed by the named entity. Other examples are "&copy;" for © and "&trade;" for ™.
